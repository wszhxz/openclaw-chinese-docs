# 贡献于 OpenClaw 威脅模型

感谢您帮助使 OpenClaw 更加安全。此威胁模型是一个活文档，我们欢迎任何人的贡献——您不需要是安全专家。

## 贡献方式

### 添加威胁

发现我们尚未涵盖的攻击向量或风险？在 [openclaw/trust](https://github.com/openclaw/trust/issues) 上打开一个问题，并用您自己的话描述它。您不需要知道任何框架或填写每个字段——只需描述场景。

**包含以下内容会很有帮助（但不是必需的）：**

- 攻击场景及其可能的利用方式
- 受影响的 OpenClaw 部分（CLI、网关、通道、ClawHub、MCP 服务器等）
- 您认为其严重程度（低 / 中 / 高 / 关键）
- 任何相关的研究链接、CVE 或现实世界示例

我们在审查期间会处理 ATLAS 映射、威胁 ID 和风险评估。如果您想包含这些详细信息，很好——但这不是预期的。

> **这是用于向威胁模型添加内容，而不是报告实时漏洞。** 如果您发现了可利用的漏洞，请参阅我们的 [信任页面](https://trust.openclaw.ai) 获取负责任披露说明。

### 建议缓解措施

有关如何解决现有威胁的想法？打开一个引用该威胁的问题或 PR。有用的缓解措施是具体且可操作的——例如，“网关处每发送者每分钟 10 条消息的速率限制”比“实现速率限制”更好。

### 提出攻击链

攻击链展示了多个威胁如何组合成一个现实的攻击场景。如果您看到危险的组合，请描述步骤以及攻击者如何将它们串联起来。描述攻击在实践中展开的简短叙述比正式模板更有价值。

### 修复或改进现有内容

拼写错误、澄清、过时信息、更好的示例——欢迎 PR，无需问题。

## 我们使用的内容

### MITRE ATLAS

此威胁模型基于 [MITRE ATLAS](https://atlas.mitre.org/)（人工智能系统的对抗性威胁景观），一个专门为 AI/ML 威胁（如提示注入、工具滥用和代理利用）设计的框架。您不需要了解 ATLAS 即可贡献——我们在审查期间将提交内容映射到框架中。

### 威胁 ID

每个威胁都会获得一个 ID，如 `T-EXEC-003`。类别包括：

| 代码    | 类别                                   |
| ------- | ------------------------------------------ |
| RECON   | 侦察 - 信息收集     |
| ACCESS  | 初始访问 - 获取入口             |
| EXEC    | 执行 - 运行恶意操作      |
| PERSIST | 持久化 - 维持访问           |
| EVADE   | 防御规避 - 避免检测       |
| DISC    | 发现 - 了解环境 |
| EXFIL   | 数据泄露 - 盗窃数据               |
| IMPACT  | 影响 - 损害或中断              |

ID 在审查期间由维护人员分配。您不需要选择一个。

### 风险级别

| 级别        | 含义                                                           |
| ------------ | ----------------------------------------------------------------- |
| **关键** | 完全系统沦陷，或高可能性 + 关键影响      |
| **高**     | 可能造成重大损害，或中等可能性 + 关键影响 |
| **中**   | 中等风险，或低可能性 + 高影响                    |
| **低**      | 不太可能且影响有限                                       |

如果您不确定风险级别，只需描述影响，我们将进行评估。

## 审查流程

1. **分类** - 我们在 48 小时内审查新提交
2. **评估** - 我们验证可行性，分配 ATLAS 映射和威胁 ID，验证风险级别
3. **文档** - 我们确保一切格式正确且完整
4. **合并** - 添加到威胁模型和可视化
