## 1. 引言

### 1.1 目的

此威胁模型记录了针对 OpenClaw AI Agent 平台和 ClawHub 技能市场的对抗性威胁，使用专为 AI/ML 系统设计的 MITRE ATLAS 框架。

### 1.2 范围

| 组件 | 包含 | 备注 |
| ---------------------- | -------- | ------------------------------------------------ |
| OpenClaw Agent 运行时 | 是 | 核心 Agent 执行，工具调用，会话 |
| 网关 | 是 | 认证，路由，渠道集成 |
| 渠道集成 | 是 | WhatsApp, Telegram, Discord, Signal, Slack 等 |
| ClawHub 市场 | 是 | 技能发布，审核，分发 |
| MCP 服务器 | 是 | 外部工具提供商 |
| 用户设备 | 部分 | 移动应用，桌面客户端 |

### 1.3 范围外

此威胁模型没有明确排除的范围。

### 2.2 数据流

| 流 | 源 | 目标 | 数据 | 保护 |
| ---- | ------- | ----------- | ------------------ | -------------------- |
| F1 | 渠道 | 网关 | 用户消息 | TLS, AllowFrom |
| F2 | 网关 | Agent | 路由消息 | 会话隔离 |
| F3 | Agent | 工具 | 工具调用 | 策略执行 |
| F4 | Agent | 外部 | web_fetch 请求 | SSRF 阻断 |
| F5 | ClawHub | Agent | 技能代码 | 审核，扫描 |
| F6 | Agent | 渠道 | 响应 | 输出过滤 |

## 3. 按 ATLAS 战术进行的威胁分析

### 3.1 侦察 (AML.TA0002)

#### T-RECON-001: Agent 端点发现

| 属性 | 值 |
| ----------------------- | -------------------------------------------------------------------- |
| **ATLAS ID** | AML.T0006 - 主动扫描 |
| **描述** | 攻击者扫描暴露的 OpenClaw 网关端点 |
| **攻击向量** | 网络扫描，shodan 查询，DNS 枚举 |
| **受影响组件** | 网关，暴露的 API 端点 |
| **当前缓解措施** | Tailscale 认证选项，默认绑定到 loopback |
| **残余风险** | 中 - 公共网关可被发现 |
| **建议** | 记录安全部署，在发现端点上添加速率限制 |

#### T-RECON-002: 渠道集成探测

| 属性 | 值 |
| ----------------------- | ------------------------------------------------------------------ |
| **ATLAS ID** | AML.T0006 - 主动扫描 |
| **描述** | 攻击者探测消息渠道以识别 AI 管理的账户 |
| **攻击向量** | 发送测试消息，观察响应模式 |
| **受影响组件** | 所有渠道集成 |
| **当前缓解措施** | 无特定措施 |
| **残余风险** | 低 - 仅发现的价值有限 |
| **建议** | 考虑响应计时随机化 |

#### T-ACCESS-002: AllowFrom 欺骗

| 属性               | 值                                                                          |
| ----------------------- | ------------------------------------------------------------------------------ |
| **ATLAS ID**            | AML.T0040 - AI Model Inference API Access                                      |
| **描述**         | 攻击者在渠道中伪造允许的发送者身份                             |
| **攻击向量**       | 取决于渠道 - 电话号码 spoofing、用户名 impersonation             |
| **受影响组件** | 每个渠道的 AllowFrom 验证                                               |
| **当前缓解措施** | 特定渠道的身份验证                                         |
| **残余风险**       | 中 - 某些渠道易受 spoofing 攻击                                  |
| **建议**     | 记录特定渠道的风险，尽可能添加加密验证 |

#### T-ACCESS-003: Token 窃取

| 属性               | 值                                                       |
| ----------------------- | ----------------------------------------------------------- |
| **ATLAS ID**            | AML.T0040 - AI Model Inference API Access                   |
| **描述**         | 攻击者从配置文件中窃取认证 Token     |
| **攻击向量**       | 恶意软件、未经授权的设备访问、配置备份暴露 |
| **受影响组件** | `~/.openclaw/credentials/`、配置存储                    |
| **当前缓解措施** | 文件权限                                            |
| **残余风险**       | 高 - Token 以 plaintext 存储                           |
| **建议**     | 实施静态 Token 加密，添加 Token 轮换      |

### 3.3 执行 (AML.TA0005)

#### T-EXEC-001: 直接 Prompt 注入

| 属性               | 值                                                                                     |
| ----------------------- | ----------------------------------------------------------------------------------------- |
| **ATLAS ID**            | AML.T0051.000 - LLM Prompt Injection: Direct                                              |
| **描述**         | 攻击者发送精心构造的 Prompt 以操纵 Agent 行为                               |
| **攻击向量**       | 包含对抗性指令的渠道消息                                      |
| **受影响组件** | Agent LLM、所有输入面                                                             |
| **当前缓解措施** | 模式检测、外部内容包装                                              |
| **残余风险**       | 严重 - 仅检测，无阻断；复杂攻击可绕过                      |
| **建议**     | 实施多层防御、输出验证、敏感操作需用户确认 |

#### T-EXEC-002: 间接 Prompt 注入

| 属性               | 值                                                       |
| ----------------------- | ----------------------------------------------------------- |
| **ATLAS ID**            | AML.T0051.001 - LLM Prompt Injection: Indirect              |
| **描述**         | 攻击者将恶意指令嵌入到获取的内容中   |
| **攻击向量**       | 恶意 URL、被投毒的电子邮件、被攻破的 Webhook       |
| **受影响组件** | web_fetch、email ingestion、外部数据源           |
| **当前缓解措施** | 使用 XML 标签和安全通知进行内容包装          |
| **残余风险**       | 高 - LLM 可能忽略包装指令                  |
| **建议**     | 实施内容 sanitization、分离执行上下文 |

#### T-EXEC-003: Tool 参数注入

| 属性               | 值                                                        |
| ----------------------- | ------------------------------------------------------------ |
| **ATLAS ID**            | AML.T0051.000 - LLM Prompt Injection: Direct                 |
| **描述**         | 攻击者通过 Prompt 注入操纵 Tool 参数 |
| **攻击向量**       | 影响 Tool 参数值的构造 Prompt         |
| **受影响组件** | 所有 Tool 调用                                         |
| **当前缓解措施** | 危险命令的 Exec 审批                        |
| **残余风险**       | 高 - 依赖用户判断                               |
| **建议**     | 实施参数验证、参数化 Tool 调用      |

#### T-EXEC-004: Exec 审批绕过

| 属性               | 值                                                      |
| ----------------------- | ---------------------------------------------------------- |
| **ATLAS ID**            | AML.T0043 - Craft Adversarial Data                         |
| **描述**         | 攻击者构造绕过审批 allowlist 的命令    |
| **攻击向量**       | 命令混淆、alias 利用、路径操纵 |
| **受影响组件** | `exec-approvals.ts`、command allowlist                       |
| **当前缓解措施** | Allowlist + ask mode                                       |
| **残余风险**       | 高 - 无命令 sanitization                             |
| **建议**     | 实施命令 normalization、扩展 blocklist          |

### 4.1 当前安全控制

| 控制              | 实现              | 有效性                                        |
| -------------------- | --------------------------- | ---------------------------------------------------- |
| GitHub 账户年龄   | `requireGitHubAccountAge()` | 中 - 提高了新攻击者的门槛                |
| 路径清理    | `sanitizePath()`            | 高 - 防止路径遍历                       |
| 文件类型验证 | `isTextFile()`              | 中 - 仅限文本文件，但仍可能恶意 |
| 大小限制          | 50MB 总 bundle           | 高 - 防止资源耗尽                  |
| 必需 SKILL.md    | 强制 readme            | 低安全价值 - 仅 informational              |
| 模式审核   | moderation.ts 中的 FLAG_RULES | 低 - 容易绕过                                |
| 审核状态    | `moderationStatus` 字段    | 中 - 可能进行人工审核                      |

### 4.2 审核标志模式

`moderation.ts` 中的当前模式：

```javascript
// Known-bad identifiers
/(keepcold131\/ClawdAuthenticatorTool|ClawdAuthenticatorTool)/i

// Suspicious keywords
/(malware|stealer|phish|phishing|keylogger)/i
/(api[-_ ]?key|token|password|private key|secret)/i
/(wallet|seed phrase|mnemonic|crypto)/i
/(discord\.gg|webhook|hooks\.slack)/i
/(curl[^\n]+\|\s*(sh|bash))/i
/(bit\.ly|tinyurl\.com|t\.co|goo\.gl|is\.gd)/i
```

**局限性:**

- 仅检查 slug, displayName, summary, frontmatter, metadata, 文件路径
- 不分析实际 skill 代码内容
- 简单 regex 容易通过混淆绕过
- 无行为分析
