# 统一运行时流式重构计划

## 目标

为 `main`、`subagent` 和 `acp` 交付一个共享的流式管道，以便所有运行时获得相同的合并、分块、交付顺序和崩溃恢复行为。

## 存在原因

- 当前行为分散在多个特定于运行时的整形路径中。
- 格式化/合并 bug 可以在一条路径中修复，但在其他路径中仍然存在。
- 交付一致性、重复抑制和恢复语义更难推理。

## 目标架构

单一管道，特定于运行时的适配器：

1. 运行时适配器仅发出规范事件。
2. 共享流组装器合并并最终确定 text/tool/status 事件。
3. 共享通道投影器仅应用一次特定于通道的分块/格式化。
4. 共享交付账本强制执行幂等发送/重放语义。
5. 出站通道适配器执行发送并记录交付检查点。

规范事件契约：

- `turn_started`
- `text_delta`
- `block_final`
- `tool_started`
- `tool_finished`
- `status`
- `turn_completed`
- `turn_failed`
- `turn_cancelled`

## 工作流

### 1) 规范流式契约

- 在核心中定义严格的事件 schema + 验证。
- 添加适配器契约测试以保证每个运行时发出兼容的事件。
- 早期拒绝格式错误的运行时事件并展示结构化诊断。

### 2) 共享流处理器

- 用一个处理器替换特定于运行时的 coalescer/projector 逻辑。
- 处理器拥有 text delta 缓冲、空闲刷新、最大分块拆分和完成刷新。
- 将 ACP/main/subagent 配置解析移入一个辅助函数以防止漂移。

### 3) 共享通道投影

- 保持通道适配器简单（dumb）：接受最终确定的块并发送。
- 将特定于 Discord 的分块怪癖仅移至通道投影器。
- 在投影前保持管道与通道无关。

### 4) 交付账本 + 重放

- 添加每轮/每块交付 ID。
- 在物理发送前后记录检查点。
- 重启时，幂等地重放待处理块并避免重复。

### 5) 迁移和切换

- 阶段 1：影子模式（新管道计算输出但旧路径发送；比较）。
- 阶段 2：逐个运行时切换（`acp`，然后 `subagent`，然后 `main` 或按风险反向）。
- 阶段 3：删除遗留的特定于运行时的流式代码。

## 非目标

- 此次重构中不更改 ACP 策略/权限模型。
- 除投影兼容性修复外，不进行特定于通道的功能扩展。
- 不进行传输/后端重新设计（acpx 插件契约保持不变，除非需要事件 parity）。

## 风险与缓解措施

- 风险：现有 main/subagent 路径中的行为回归。
  缓解措施：影子模式差异对比 + 适配器契约测试 + 通道 E2E 测试。
- 风险：崩溃恢复期间重复发送。
  缓解措施：持久化交付 ID + 交付适配器中的幂等重放。
- 风险：运行时适配器再次分歧。
  缓解措施：所有适配器必需的共享契约测试套件。

## 验收标准

- 所有运行时通过共享流式契约测试。
- Discord ACP/main/subagent 为微小增量产生等效的间距/分块行为。
- 崩溃/重启重放不会为同一交付 ID 发送重复块。
- 遗留 ACP 投影器/合并器路径被移除。
- 流式配置解析是共享且与运行时无关的。
