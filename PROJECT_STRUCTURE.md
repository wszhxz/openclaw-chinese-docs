# OpenClaw 中文文档项目结构说明

## 项目设计逻辑

本项目采用双分支结构实现中英文文档的同步维护：

### 分支说明

#### `original-en` 分支
- **目的**：同步原 OpenClaw 项目的英文文档
- **内容**：完整的英文文档网站
- **更新机制**：定期与上游项目同步，获取最新英文文档
- **用途**：作为中文翻译的参考源

#### `main` 分支  
- **目的**：存放中文翻译文档
- **内容**：GitHub Pages 网站的中文版本
- **docs 目录**：中文翻译文档的根目录
- **更新机制**：基于 `original-en` 分支的变更进行翻译更新

### 工作流程

1. **同步更新**：上游英文文档更新时，同步到 `original-en` 分支
2. **差异检测**：比较 `original-en` 与 `main` 分支的文档差异
3. **翻译更新**：将新增或修改的英文内容翻译为中文
4. **部署发布**：中文文档通过 GitHub Pages 发布

### 目录结构

```
main 分支:
├── docs/                 # GitHub Pages 根目录，存放中文文档
│   ├── index.md         # 中文首页
│   └── ...              # 其他中文翻译文档
├── scripts/             # 自动化脚本
├── .github/workflows/   # CI/CD 工作流
└── ...

original-en 分支:
├── docs/                # 英文原始文档
│   ├── index.md        # 英文首页
│   └── ...             # 其他英文文档
└── ...
```

### 维护策略

- 通过自动化脚本监控 `original-en` 分支变化
- 及时将英文更新翻译为中文
- 保持中英文文档的一致性和时效性
